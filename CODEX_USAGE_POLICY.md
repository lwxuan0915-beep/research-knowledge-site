# Codex 使用策略

本文件用于降低 Codex 额度消耗，规范第二阶段文章精修中的轻量执行方式。

## 默认模式

如无特别说明，Codex 默认采用轻量执行模式。

- ChatGPT 负责知识内容、文章逻辑、任务拆分和修改方案设计。
- Codex 只负责最小文件替换、少量配置修改和必要构建验证。
- 普通 Markdown 文章修改默认不运行 build。
- 只有修改配置、侧边栏、导航、package.json、目录结构、部署相关内容，或进行构建失败排错时，才运行 build。
- 轻量模式输出不超过 8 行。
- 单次轻量任务只修改 1-3 个文件。
- 不得顺手修改无关文件。

## Prompt 模板

替换单篇文章：

```text
请进入轻量执行模式。只替换 docs/模块/文章.md 的正文内容，不修改其他文件，不运行 build。完成后只输出修改文件和遗留问题。
```

修改侧边栏：

```text
请进入轻量执行模式。只修改当前生效的 VitePress 配置，把指定文章加入对应侧边栏，不改变一级目录顺序。完成后运行 npm.cmd run build，并只输出修改文件、build 结果和遗留问题。
```
